# Change summary schema

Start with a short table of contents for all five outputs:

| Output | What changed | Reviewer note |
|--------|--------------|---------------|
| `gaps.md` | Plain-language tier, verdict, matches, gaps, bridges | Highlight hard gaps and the recommended next step |
| `tailored-resume.md` | Resume view from verified facts | Note strongest edits |
| `suggested-additions.md` | Verify-before-use questions for you | Confirm suggestions are gated |
| `cover-letter.md` | Standard-structure cover letter (header + 3-4 paragraphs + sign-off) | Confirm facts trace to `resume.md` |
| `change-summary.md` | This review guide | Keep concise |

Then document tailored-resume edits:

| Field | Values |
|-------|--------|
| Location | Section + employer or heading |
| Type | `rephrase` \| `reorder` \| `emphasize` \| `trim` |
| Before | Short quote from base |
| After | Short quote from tailored |
| Job reason | Requirement or keyword driving change |
| Suggested emphasis | Optional tag when reframing without new facts |

Omit changes that only fix typos unless job-relevant.

End with:

- Suggested-additions gate: confirm suggestions are not treated as facts.
- Do-not-claim audit: list hard job gaps that stayed out of `tailored-resume.md`.
- Plaintext export note when relevant.
