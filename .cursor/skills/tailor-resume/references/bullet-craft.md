# Bullet craft

Use one frame per bullet; choose the one that fits the available facts.

| Frame | Pattern | Use when |
|-------|---------|----------|
| CAR | Challenge -> Action -> Result | The source fact names a problem and outcome |
| STAR | Situation -> Task -> Action -> Result | The output needs a clear talking-point story |
| XYZ | Accomplished X by doing Y, measured by Z | The source fact has a clear metric |

Action verbs by activity:

- Build: built, implemented, delivered, launched, integrated
- Ship: released, rolled out, migrated, introduced, automated
- Scale: optimized, hardened, expanded, stabilized, monitored
- Improve: reduced, clarified, simplified, consolidated, remediated
- Lead: led, mentored, coordinated, owned, guided
- Partner: collaborated, aligned, translated, supported, reviewed

Quantification axes:

- Latency, throughput, error rate, uptime, incident count
- Adoption, monthly active users, request volume, queue depth
- Time saved, release cadence, manual steps removed
- Revenue, cost, refunds, credits, invoice accuracy
- Team size, reviewer count, stakeholder groups

So-what test:

- Name the system or workflow, not just the technology.
- State the user, business, reliability, or delivery impact.
- Keep metrics exactly as `resume.md` provides them; do not invent proxies.
- Prefer one concrete accomplishment over a list of tools.
- Recomposition is allowed when every claim still maps to `resume.md`.
