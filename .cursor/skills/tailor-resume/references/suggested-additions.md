# Suggested additions output

Write `suggested-additions.md` as a check-before-using checklist.

Lead with this warning: these are proposed additions, not facts. You need to confirm each item before adding it to `resume.md`, `tailored-resume.md`, or an application.

For each entry, use this schema:

## Short suggestion title

**Question for you:** A direct yes/no question anchored to a specific `resume.md` fact.

**Why this matters for this job:** One sentence connecting the missing detail to a job requirement.

**Proposed wording if true:** One resume bullet or talking point. Use brackets for unknown metrics rather than inventing them.

**Where it would live:** The employer, section, or output that would use it after verification.

**Verification cue:** What to check: code path, dashboard, incident, metric, stakeholder, PR, design doc, or memory of the work.

Do not include suggestions without a related fact in `resume.md`.
