# Master profile contract

`resume.md` is the master profile.

- Anything in the job description that cannot be traced to `resume.md` -> **Do not claim** section only.
- Tailored resume is a **view** of the master profile for one role, not a second truth source.
- Do not delete rare but true skills from base—only de-emphasize in tailored file if irrelevant.
- Suggestions in `suggested-additions.md` are not part of the master profile. They become eligible facts only after the user verifies them and updates `resume.md`.
- Until `resume.md` changes, downstream outputs may discuss suggestions only as questions, never as known facts.
